# ДЗ 5 Описание

 - Базовый агент `api/BaseFetchAgent.ts` был взят из лайфкодинга урока и доработан. Теперь заголовок Content-Type будет установлен в application/json только в случае, если он не был явно передан в конфиге. А если конфиг был передан с объектом headers но без этого заголовка, он перезаписан не будет.
 - В `types.ts` описан дополнительный тип ApiError и его Type Guard. Они используются в BaseFetchAgent для получения и проброса сообщений об ошибке, полученных от сервера, а не просто статус кода.
 - В TasksFetchAgent базовый url не захардкожен, а передается параметром. Реализует все CRUD методы: `getTasks()`, `createTask(task)`, `getTask(id)`, `updateTask(id, task)`, `deleteTask(id)`.
 - Интерфейс был взят из прошлой работы и адаптирован под TS. Архитектура чуть упрощена.
